Where Path A and Path B diverge — by design

Path A — Open Platform

The owner can export the files.

The legal question is whether the custodian has the right to access, deploy, and reconstruct. This form IS the Tier-2 "other record" under §871 — it grants that right in writing.

Simpler consent block. Self-ownership attestation replaces the platform legacy-tool question. Custodian technical-competency tier drives whether the plan includes deployment instructions.

Path B — Walled Garden

The owner has no file access.

The platform's TOS stands between owner and agent. The plan must survive TOS constraints and satisfy statutory consent triggers.

Carries the §873 content-disclosure consent, the SCA §2701 federal dovetail, and a legacy-tool triage (Tier 1 / Tier 2 / Tier 3 / Unsure) that drives which tier the plan lands in. Platform-specific TOS lookup is your work product.

Both forms carry

  • Age / sound-mind attestation (§1550)
  • Fiduciary-duty acknowledgment (§880)
  • UETA ToS clickwrap (§1633.1 et seq.)
  • Commercially-reasonable-efforts disclosure — pharmaceutical label, no guarantees
  • Custodian Reconstruction Clause boilerplate for the plan document
  • E-signature validation — typed name must match typed owner legal name (enforced client-side)
  • California choice-of-law — V1; other states gated on your cross-check

Data minimality — CCPA / CPRA posture

Every field was audited against "could this hit SPI under CPRA?" The rules we ship with:

  • Never asks for passwords, PINs, API keys, 2FA backup codes, or credentials of any kind. Helper text says so explicitly at every field that could be misread.
  • Financial access is a Yes/No. The descriptive text is conditional and demands descriptions only ("a trading account," "a crypto wallet"), never credentials.
  • File locations are captured as paths or location names, not uploads.
  • Soul Interview worksheet is referenced by link or attached to email directly — never uploaded through the form.
  • Submission routing is mailto:charles@thelewfirm.com + a local text download. No third-party form processor holds PII.

Submission flow

  1. Owner fills the form in-browser.
  2. Client-side validation (required fields + e-signature match against owner legal name).
  3. On submit:
    • A plain-text intake summary downloads to the owner's device (wp-intake-v[1|2]-<name>-<date>.txt)
    • The owner's mail client opens pre-addressed to charles@thelewfirm.com, subject and body pre-filled
    • The owner attaches the downloaded file and sends
    • localStorage draft is cleared
  4. A "Save Draft (Local)" button preserves progress in the owner's browser only — no server storage.

Known weak point: the mailto handoff is fine for pre-intake, not for a signed plan. Phase 2 migration recommended: DocuSign or HelloSign for the plan document itself so the executed instrument has an auditable e-sign trail. (This was flagged in the 2026-04-13 self-study and is tracked for a future pass.)

What I want your red pen on

  1. §873 consent language (Path B Section 05, checkbox 2). Is the phrasing tight enough to satisfy both the statute and the SCA dovetail? Or should it split into two checkboxes — one for each?
  2. Custodian reconstruction grant (Path A Section 06, checkbox 2). Is a checkbox sufficient to grant the right in the intake, or does this belong migrated up into the plan document itself with a separately executed grant?
  3. Commercially-reasonable-efforts disclosure. Is the wording defensible if a customer later claims WP "promised" to preserve their Replika account? If not, I'll tighten.
  4. Choice-of-law — California-only gate. Should the Owner's State of Residence field block submission on non-CA residents in V1, or let them through with a "California law governs" notice? Currently: accepts any state, notes CA governs.
  5. Fiduciary-duty acknowledgment (§880). Should this also appear in the plan document itself, or is the intake-level acknowledgment sufficient?
  6. Age gate. Checkbox only. Should we add a typed DOB for stronger evidence, or does that open CCPA SPI exposure?
  7. Anything I missed. This is my best self-study effort from the 2026-04-13 prep. You are the bar-licensed filter — red-pen anything that should escalate from intake to plan document, or any statute I cited incorrectly.
