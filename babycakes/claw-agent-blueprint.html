The Interview Framework

Same engine we built for Worth Protecting — adapted for law firms

Engine Built & Proven

Charles already knows the Soul Interview from Worth Protecting. That technology — the structured questioning system that captures who someone really is — is ours. We built it. We've run 77 deep questions through it in production. It captures nuance, not just checkboxes. The difference between "what area of law do you practice" and "what's the mistake that would make you fire a first-year associate" is the difference between a form and an interview. Our system does the second kind.

The Firm Interview is the same engine with a legal-specific question set. A senior partner sits down for 30-45 minutes. By the end, the AI agent knows that firm the way a senior associate would after six months — writing voice, risk tolerance, client expectations, the institutional knowledge that only lives in people's heads. And unlike a senior associate, it never forgets any of it.

We have the engine. Charles and the Ninja team have the legal substance. Together, that's a product nobody else can build — because nobody else has both halves.

How the Soul Interview works (what we've already built)

  • Structured questioning: Not open-ended chat — carefully ordered questions that build on each other, each one designed to capture a specific dimension of identity
  • Answer processing: Raw answers get structured into a profile document that the AI agent reads before every interaction — like a permanent employee handbook
  • Living document: The profile updates over time as the firm evolves. New partner? Update the profile. Changed approach to risk? Update the profile. Agent adapts immediately.
  • Proven at scale: Our production interview system has processed 77 detailed questions with branching follow-ups. It captures nuance, not just facts.

What makes this different from "just asking the AI to remember"

  • Generic AI memory stores random things you've mentioned. Our interview is designed — every question has a purpose, every answer maps to a behavior the agent will exhibit.
  • The interview captures things a firm wouldn't think to tell an AI — "what mistake would make you fire a first-year?" teaches the agent what NOT to do. "What does good work look like HERE?" sets the quality bar in the firm's own words.
  • A firm fills it out once. The agent carries it forever. No repeated prompting, no "remind the AI how we do things."

The Memory System

Not a feature. The backbone.

Built & In Daily Production

Here's what separates this from every other AI legal tool on the market: the agent gets smarter the longer you use it. Not in a marketing-slide way. In a real, measurable, "it remembered that I corrected it three weeks ago and never made the same mistake again" way.

Our memory system has been in daily production for months. Not a demo. Not a proof-of-concept. Production infrastructure that we depend on every single day, handling hundreds of structured memories with instant semantic search. This is the same technology that would power every CLAW Agent — and it's already built.

Structured Storage

Memory isn't a pile of old chats. Every memory is typed, tagged, and organized — corrections, preferences, client knowledge, firm patterns. The agent knows WHAT it knows and WHERE it learned it.

Semantic Search

Powered by vector embeddings (ChromaDB + sentence-transformers). The agent searches by meaning, not keywords. Ask about "indemnification preferences" and it finds relevant memories even if those exact words were never used.

Persistent Across Sessions

Close the window. Come back next week. The agent remembers everything. Not just what you said — who you are, what you prefer, what corrections you've made, what patterns your firm follows.

Self-Correcting

"We never structure deals this way." The agent remembers that correction forever. Monday's mistake becomes Tuesday's correct answer. It doesn't repeat errors — it learns from every interaction.

Proactive Recall

The agent doesn't wait to be reminded. Before every response, it automatically searches its memory for relevant context. Working on Acme Corp? It already pulled up everything it knows about Acme.

Indexed at Scale

Hundreds of memory entries, searchable instantly. No slowdown over time. The agent at month 12 is faster than at month 1 because it has more knowledge to draw from, not more weight to carry.

What the memory actually looks like (real architecture)

  • Memory files: Structured markdown documents organized by type — each with metadata (name, description, type, date, importance score)
  • Vector database: ChromaDB indexes every memory file with sentence-transformer embeddings for semantic search
  • MCP server: A dedicated memory server that the agent queries through a standard protocol — search, filter by type, list, get statistics
  • Automatic indexing: New memories are indexed incrementally. Full rebuild available when needed. Search works in milliseconds.
  • Memory types we've defined: User profiles, feedback/corrections, project context, reference pointers, entity knowledge graphs, diary entries — each type has different storage and retrieval logic

How this translates to a law firm

  • Correction memory: "Our standard NDA always includes a 2-year non-compete" — the agent defaults to this going forward
  • Client memory: "Acme Corp's GC prefers bullet points over paragraphs" — remembered across all Acme matters
  • Firm pattern memory: "We never use liquidated damages clauses under $50K" — becomes a rule the agent follows
  • Matter memory: "The Smith case has a Delaware jurisdiction complication" — context carries across every session on that matter
  • Feedback memory: "Last time you flagged too many low-risk items" — sensitivity auto-adjusts

Honest comparison to what's out there

  • ChatGPT and Claude both have memory features now. But theirs is flat — a list of things you've mentioned, with no structure, limited capacity, and no semantic search.
  • Legal AI startups (Harvey, CoCounsel) have their own persistence, but it's locked to their platform and their model. Switch providers, lose everything.
  • Our system is structured, searchable, model-agnostic, and built around firm identity. Most memory systems remember what you SAID. Ours remembers who you ARE.
  • We're not claiming we invented AI memory. We're claiming ours is deeper, more structured, and portable — and we can prove it because it's been running in production for months.

Phase 2: Custom Training

This is where it goes from good to unreplicable

Infrastructure Partner Identified

Phase 1 — the Core framework — gives every firm a CLAW-trained agent that follows the methodology through instructions. It reads the rulebook before every interaction. That's powerful. But there's a ceiling: the agent is following instructions. It's reading CLAW, not thinking CLAW.

Phase 2 removes that ceiling.

Custom Training means we take a firm's own historical documents — their last 5 to 10 years of contracts, memos, briefs, correspondence — and we train a model directly on that work. Not prompting. Not instructing. Actual fine-tuning. The neural weights shift. The model doesn't read how the firm writes — it writes that way natively. It doesn't check a risk tolerance setting — it has risk intuition shaped by thousands of real decisions that firm actually made.

The difference between Phase 1 and Phase 2 is the difference between giving someone a recipe and teaching them to cook. Phase 1 follows the recipe perfectly. Phase 2 improvises because it understands the cuisine.

How it works

  • Firm provides historical documents: Contracts, memos, briefs, client communications — anonymized and securely transferred. The more data, the better the training.
  • We fine-tune on dedicated GPU servers: We have an infrastructure partner with enterprise GPU capacity (SeqPU). Not shared cloud instances — dedicated hardware for secure legal workloads.
  • The model learns the firm's patterns: Writing style, clause preferences, risk assessment patterns, negotiation tendencies — absorbed from real work product, not described in a profile.
  • Monthly retraining: As the firm produces new work, the model gets retrained periodically. It doesn't just remember the past — it evolves with the firm.
  • HITL Score verified after every training cycle: Each retrained model gets re-evaluated. Training doesn't just make it different — it has to make it better, confirmed by HITL Score metrics.

Why this is a different tier of product

  • Instruction-following has limits. You can tell an AI "be conservative with risk flags" — but a model trained on 10,000 real contracts from a conservative firm doesn't need to be told. It IS conservative. The behavior is baked in, not bolted on.
  • Competitors can't touch this. Harvey and CoCounsel use one model for all customers. They can't fine-tune per firm — their architecture doesn't support it. We can, because our framework is model-agnostic and our GPU partner provides dedicated training infrastructure.
  • The moat deepens over time. Every month of training makes the firm's agent more specialized, more accurate, more irreplaceable. After a year of custom training, switching to a competitor means starting from zero. That's retention built into the product.
  • Premium pricing, justified. $2K-5K setup plus $500/month — but the value is proportional. A firm paying $500/month for an agent that thinks like their best senior associate? That associate costs $15K/month in salary.

The GPU infrastructure

  • Our partner operates SeqPU — enterprise GPU rental infrastructure built for AI workloads
  • Dedicated hardware, not shared cloud — critical for legal data security
  • Capacity to train multiple firm models in parallel as the customer base scales
  • We've already made contact and the relationship is in development
  • This is the "workshop" — the framework is the car, CLAW is the GPS, and the GPU servers are the factory where custom engines get built

The honest version

  • Phase 2 is not ready today. Phase 1 launches first. Custom Training is the premium tier we build once the Core framework is proven.
  • Fine-tuning requires meaningful data volume — firms with 5+ years of documents benefit most. Smaller firms may not have enough data to justify it.
  • GPU costs eat into margins (~60% vs ~90% on Core) but the price point is higher and the retention is stronger.
  • The GPU partnership is identified but not yet formalized — we're early in that relationship.
