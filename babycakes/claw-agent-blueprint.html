01

What Is This Thing?

The short version

It's a wrapper that goes around any AI model (Claude, GPT, Llama, etc.) and turns it into a legal professional that already knows the CLAW method and already knows your firm.

Think of it like this: the AI model is a brain. A very smart brain, but it has no training, no memory of your firm, no knowledge of CLAW, and amnesia every time you close the window. Our framework gives that brain everything it's missing — methodology, memory, firm knowledge, document workflows, and security.

The lawyer doesn't interact with the AI model directly. They interact with the framework, which uses the model as its engine underneath. They can swap the engine anytime without losing anything.

The CLAW Method Layer

The methodology baked into the agent's instructions

Needs Charles

This is the core differentiator. Every time the agent reviews a contract, drafts a memo, or flags a risk, it follows the CLAW method automatically. The lawyer doesn't prompt it to use CLAW — it just does, because the method is encoded into the framework's instruction layer.

Technically, this is a structured set of rules, decision trees, and analysis steps that get loaded before every interaction. Think of it as the agent's training manual that it re-reads before every task.

What we need from Charles to build this

  • The formal CLAW method steps — what's the process from start to finish?
  • Decision trees — when reviewing a contract, what does the agent check first? Second? What triggers a red flag vs. a yellow flag?
  • Output format — what does a CLAW analysis look like when it's done right? Show us 3-5 examples.
  • Edge cases — where do junior lawyers get CLAW wrong? Those mistakes become our test cases.
  • Terminology — any CLAW-specific terms or categories the agent needs to know.

What we handle

  • Encoding the method into structured agent instructions
  • Building test cases to verify the agent follows CLAW correctly
  • Version control — when CLAW evolves, we update the framework

A structured onboarding that teaches the agent who the firm is

Framework Built — Questions Need Legal Input

We've already built a working interview system. It's been tested extensively (77 questions, proven in production). The technology works. What we need is the legal-specific question set — designed with Charles — that captures what makes each firm unique.

The interview takes 30-45 minutes with a senior partner. Their answers get structured into a profile that the agent reads before every interaction. The agent doesn't just know law — it knows THIS firm's law.

The Memory System

Persistent knowledge that survives between conversations

Built & Proven

This is built. It works. It's been in daily production use for months.

Most AI tools remember your conversation while the window is open and forget everything when you close it. Our memory system stores structured knowledge in a searchable database that persists forever. The agent gets smarter the more you use it.

What the memory system actually stores

  • Corrections: "We never structure indemnification clauses this way" — the agent remembers and never does it again
  • Client preferences: "Acme Corp's GC prefers bullet points over paragraphs" — remembered across all matters
  • Firm patterns: "Our standard NDA always includes a 2-year non-compete" — becomes the default
  • Matter context: "The Smith case has a complication with jurisdiction in Delaware" — carries across sessions
  • Feedback: "Last time you flagged too many low-risk items" — adjusts sensitivity going forward

How it works technically

  • Structured memory files organized by type (firm knowledge, client data, corrections, preferences)
  • Semantic search powered by vector embeddings — the agent finds relevant memories by meaning, not just keywords
  • Memory is searched automatically before every response — the agent checks what it knows before it speaks
  • Indexed and searchable across thousands of entries without slowing down

Honest comparison to competitors

  • ChatGPT and Claude now have basic memory — they remember things you've told them. But it's flat, unsearchable, and limited in size.
  • Harvey and CoCounsel have their own persistence, but it's locked to their platform and their model.
  • Our memory is structured, searchable, model-agnostic, and built around firm identity — not just conversation history.
  • The real difference: most memory systems remember what you SAID. Ours remembers who you ARE.

Document Workflows

Structured pipelines for common legal tasks

Needs Build — 4-6 Weeks

Instead of "ask the AI a question and hope for the best," workflows are structured step-by-step processes that produce consistent output. The lawyer uploads a contract, the workflow runs the CLAW analysis, flags risks at the firm's threshold, and produces a review memo in the firm's format.

Phase 1 workflows (launch)

  • Contract Review: Upload a contract → CLAW analysis → risk flags (red/yellow/green) → review memo in firm format
  • Memo Draft: Input key facts and analysis → structured memo with proper citations → ready for partner review
  • Client Communication: Draft client-facing emails and letters in the firm's voice and style

Phase 2 workflows (after launch, based on demand)

  • Due Diligence Checklist: Generate and track checklist items for transactions
  • Clause Comparison: Compare specific clauses across multiple agreements
  • Regulatory Check: Flag compliance issues in specific jurisdictions

What we need from Charles

  • What does a CLAW contract review look like start to finish? Walk us through one.
  • Sample inputs and outputs — anonymized real work that shows "this is what goes in, this is what should come out"
  • Which workflows matter most? We build those first.

Security & Access Control

Ethical compliance built into the architecture

Needs Build — Design Ready

Law firms can't use AI that lets Client A's data bleed into Client B's work. This isn't optional — it's an ethical requirement. The security layer handles this at the framework level so individual lawyers don't have to think about it.

What the security layer does

  • Client matter isolation: Each client matter exists in its own walled space. The agent working on Matter A cannot access Matter B's documents or memory.
  • Role-based access: Partners see everything. Associates see their assigned matters. Paralegals see what they need. Configurable per firm.
  • Audit trail: Every interaction is logged — who asked what, when, what the agent produced. Required for compliance.
  • Data stays in the firm's control: The framework runs on the firm's infrastructure or their chosen cloud. We don't host their data.

Honest assessment

  • This is real engineering work. Not the hardest part, but not trivial.
  • Client isolation is the priority — that's the ethical firewall. Role-based access and full audit trails can come in Phase 2.
  • For the beta, we can start with single-user access per firm (one lawyer, one agent). Multi-user security is a Phase 2 feature.

Model-Agnostic Architecture

Swap the AI brain without losing anything

Architecture Proven

The framework talks to AI models through a standard interface. Claude today, GPT tomorrow, an open-source model next month — the firm's data, memory, CLAW training, and firm profile all stay the same. Only the engine changes.

Why this matters

  • AI models are improving every few months. Being locked to one provider means you're stuck when something better arrives.
  • Some firms want maximum privacy — they can run an open-source model on their own servers. No data leaves their building.
  • Some firms want maximum capability — they use Claude or GPT through an API. Their choice.
  • If a provider raises prices, gets worse, or goes away — the firm switches models in an afternoon, not a migration project.

Honest caveat

  • Different models have different strengths. Claude is better at reasoning. GPT is better at certain formats. An open-source model running locally will be slower.
  • "Swap in an afternoon" means the framework works with the new model. It doesn't mean the output quality is identical — each model needs some tuning.
  • We'll recommend models by use case and be transparent about trade-offs.

Revenue Model

Three tiers, different margins, one flywheel

Core Framework

~$500/month per firm
Full framework with CLAW method, firm interview, memory, document workflows. Firm provides their own AI model (API key). We provide the software. Margins ~90% — no compute costs on our side.

Custom Training

$2K-5K setup + $500/month
Everything in Core, plus a model fine-tuned on the firm's own historical documents (5-10 years of contracts, memos). Requires GPU compute for training. Margins ~60%.

LPEAI Bundle

Bundled with class tuition
LPEAI graduates get a Core license at graduation. Class fee increases to include it. Every class = new batch of subscribers. The distribution flywheel.

  • LPEAI teaches lawyers the CLAW method

  • Every graduating class is a new batch of subscribers
  • Subscribers pay monthly — recurring revenue

Pricing is negotiable

Why Nobody Else Can Build This

The things that are actually hard to copy

Real advantages

  • CLAW is proprietary. No other framework can offer a CLAW-trained agent because nobody else has the methodology. This is the actual moat.
  • LPEAI is the distribution channel. Every CLAW-trained lawyer needs a CLAW-capable tool. We're the only one. Built-in customer pipeline.
  • The interview technology is proven. Not theoretical — we've run hundreds of questions through it and built a working identity system. Competitors would need months to replicate this.
  • Model-agnostic means we survive provider changes. If Claude gets expensive or GPT gets better, our customers just swap. Platform-locked competitors can't say that.

Honest non-advantages (things competitors also have)

  • Basic AI memory — ChatGPT and Claude both have this now. Our memory is deeper, but we can't claim "we have memory and they don't."
  • Contract review — many AI tools can review contracts. Our edge is CLAW methodology, not the review itself.
  • Legal AI generally — Harvey, CoCounsel, and others exist. We compete on methodology + customization + model freedom, not on "we do legal AI."
